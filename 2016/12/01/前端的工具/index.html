前端的工具

Git（分布式版本控制工具）

控制工具分类：

  • 本地版本控制系统
  • 集中式版本控制系统SVN
  • 分布式版本控制系统Git
  • ​

安装：

  • [x] Window安装
  • http://git-scm.com/download/win下载Git客户端软件，和普通软件安装方式一样。
  • [x] Linux安装
  1. CentOS发行版：sudo yum install git
  2. Ubuntu发行版：sudo apt-get install git
  • [x] Mac安装
  • 打开Terminal直接输入git命令，会自动提示，按提示引导安装即可。

Git的工作原理

  1. Git管理文件的4个状态
    为追踪
    已提交
    已修改
    以暂存
  2. Git项目的3个工作区域的概念：
    Git仓库
    工作目录
    暂存区域

Git的基础操作（在本地Git仓库的操作）

  1. 配置用户
    git config –global user.name “自已的名字”
    git config –global user.email “自已的邮箱地址”
  2. 初始化仓库
    git init
  3. 查看文件状态
    git status
    git checkout 文件名 可以撤销更改的文件
  4. 添加文件到暂存区域
    git add 文件名/文件路径 “*”或者-A代表所有
  5. 提交文件到本地仓库
    git commit -m ‘添加必要性的文字’
    每一次添加文件都会生成文件快照用来删除本地文件也可以git reset –hard 快照ID返回之前的快照提交
  6. 查看提交历史
    git log
  7. 去过有修改返回第3步重新提交

Git分支操作

  1. 创建分支
    git branch 分支名称
  2. 切换分支
    git checkout 分支名称
  3. 前边两步可以用一个代码来进行执行：
    git checkout -b 分支名称 ： 创建并跳转到创建的分支
  4. 合并分支
    git merge 分支名称 ：命令用于合并指定分支到当前分支上 （此处的分支名称是被合并的分支名称）
  5. 删除分支
    git branch -d 想删除的分支名称

远程仓库的Git命令：

  1. 创建远程共享仓库：
    mkdir repo.git 创建以.git结尾目录
    cd repo.git 进入这个目录
    git init –bare 初始化一个共享仓库，也叫裸仓库 注意选项–bare
  2. 想共享仓库共享同步内容：
    git push 远程仓库地址 下载的分支
  3. 从共享仓库里去除内容：
    git clone 远程仓库的地址 远程下载分支/本地分支
  4. 向远程仓库添加本地的内容：
    git pull origin（如果origin已经关联，直接用git pull）

Git的基本的命令：

  1. git config配置本地仓库
  2. 常用git config –global user.name、git config –global user.email
  3. git config –list查看配置详情
  4. git init 初始一个仓库，添加–bare可以初始化一个共享（裸）仓库
  5. git status 可以查看当前仓库的状态
  6. git add“文件” 将工作区中的文件添加到暂存区中，其中file可是一个单独的文件，也可以是一个目录、“*”、-A
  7. git commit -m ‘备注信息’ 将暂存区的文件，提交到本地仓库
  8. git log 可以查看本地仓库的提交历史
  9. git branch查看分支
  10. git branch“分支名称” 创建一个新的分支
  11. git checkout“分支名称” 切换分支
  12. git checkout -b deeveloper 他健并切到developer分支
  13. git merge“分支名称” 合并分支
  14. git branch -d “分支名称” 删除分支
  15. git clone “仓库地址”获取已有仓库的副本
  16. git push origin “本地分支名称:远程分支名称”将本地分支推送至远程仓库，
  17. git push origin hotfix（通常的写法）相当于
  18. git push origin hotfix:hotfix
  19. git push origin hotfix:newfeature
  20. 本地仓库分支名称和远程仓库分支名称一样的情况下可以简写成一个，即git push “仓库地址” “分支名称”，如果远程仓库没有对应分支，将会自动创建
  21. git remote add “主机名称” “远程仓库地址”添加远程主机，即给远程主机起个别名，方便使用
  22. git remote 可以查看已添加的远程主机
  23. git remote show “主机名称”可以查看远程主机的信息

bower是JS下载工具

介绍：

  • 基于NodeJS的一个静态资源管理工具，由twitter公司开发维，解决大型网站中静态资源的依赖问题（依赖于NodeJS环境和git工具）。
  • bower下载插件，会下载此插件的依赖，会自动完成依赖的关系
    基本的命令：
  1. 安装bower
    npm install -g bower
  2. 查找资源信息
    bower search 查询的插件名字
  3. 安装下载资源，通过#可以指定版本
    bower install 插件名字
  4. 查看资源信息
    bower info 插件名字
  5. 卸载资源
    bower uninstall 插件名字
